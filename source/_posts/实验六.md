## 1. 实验目的

本次实验旨在掌握Linux系统的命令行基础操作，理解系统用户与文件权限的管理规则，学会配置用户级与系统级环境变量，熟悉 `/etc/passwd`、`~/.bashrc` 等核心系统配置文件的作用与修改方法，完成Linux基础运行环境的搭建与管理。通过动手实践，培养命令行操作习惯，理解Linux“一切皆文件”的设计哲学，为后续服务部署与开发环境搭建奠定基础。

## 2. 实验环境

- **操作系统**：Ubuntu Server LTS
- **运行载体**：VirtualBox 7.0.12 虚拟机
- **前置条件**：已完成实验五，使用 node1（192.168.56.102）作为操作目标
- **操作终端**：Linux系统内置Shell终端（通过SSH或虚拟机控制台）
- **用户身份**：puretaoit（具备sudo权限）
- **网络状态**：NAT可访问外网（用于安装软件包），Host-Only内网互通

### 第二部分：用户管理

#### 步骤四：查看用户与组配置文件

读取系统用户与组相关配置文件，理解用户信息存储格式：

```bash
cat /etc/passwd       # 查看所有用户账户基础信息
sudo cat /etc/shadow  # 查看用户加密密码信息（仅root可读）
cat /etc/group        # 查看系统用户组信息
id                    # 查看当前用户的UID、GID与所属组
```

`/etc/passwd` 文件格式为：`用户名:密码占位符:UID:GID:用户描述:主目录:登录Shell`

#### 步骤五：创建和管理用户

执行命令完成用户的创建、权限配置、切换与删除操作：

```bash
sudo adduser testuser           # 创建新用户testuser（交互式设置密码）
sudo passwd testuser            # 为testuser重新设置登录密码
sudo usermod -aG sudo testuser  # 将用户加入sudo组，赋予管理员权限
groups testuser                 # 查看用户所属的用户组 → testuser : testuser sudo
su - testuser                   # 切换到testuser用户
whoami                          # → testuser
sudo whoami                     # 测试sudo权限 → root（验证成功）
exit                            # 退出testuser，返回puretaoist
sudo deluser -r testuser        # 删除用户及其主目录
```

#### 步骤六：文件权限管理

通过数字法与符号法修改文件权限，调整文件的所有者与所属组：

**权限数字对照表**：

| 数字 | 权限 | 含义 |
|------|------|------|
| 7 | rwx | 读+写+执行 |
| 6 | rw- | 读+写 |
| 5 | r-x | 读+执行 |
| 4 | r-- | 只读 |
| 0 | --- | 无权限 |

**操作命令**：

```bash
# 准备测试文件
touch test.txt
ls -l test.txt  # 初始权限：-rw-rw-r--（664）

# 数字法修改权限
chmod 755 test.txt
ls -l test.txt  # 权限变更为 -rwxr-xr-x（755）

# 符号法修改权限
chmod u+x test.txt          # 为所有者增加执行权限（已存在）
chmod g-w test.txt          # 移除所属组的写权限
chmod o-r test.txt          # 移除其他用户的读权限
ls -l test.txt              # 权限变更为 -rwxr-x---（750）

# 修改文件所有者和所属组
sudo chown root:root test.txt  # 同时修改所有者和所属组为root
sudo chgrp www-data test.txt   # 仅修改所属组为www-data
ls -l test.txt                  # 查看变更结果
```

#### 步骤九：永久设置环境变量（用户级）

用户级环境变量仅对当前用户生效，三类配置文件规则如下：

| 配置文件 | 加载时机 | 作用范围 |
|----------|----------|----------|
| `~/.bashrc` | 每次打开新终端时执行（非登录Shell） | 当前用户 |
| `~/.profile` | 用户登录时执行一次（登录Shell） | 当前用户 |
| `~/.bash_profile` | 用户登录时执行（优先级高于 .profile） | 当前用户 |

编辑 `~/.bashrc` 配置永久变量：

```bash
nano ~/.bashrc
# 在文件末尾添加以下配置
export MY_CUSTOM_VAR="this is my custom var"
export PATH=$PATH:/home/puretaoist/mybin
alias ll='ls -alF'
# 保存退出（Ctrl+O, Enter, Ctrl+X）

# 执行命令立即生效（无需重新登录）
source ~/.bashrc

# 验证配置生效
echo $MY_CUSTOM_VAR   # 输出：this is my custom var
ll                    # 执行 ll 命令，等价于 ls -alF
```

#### 步骤十：永久设置环境变量（系统级）

系统级环境变量对所有用户生效，需root权限修改，三种配置方式：

**方法一：/etc/environment（全系统生效，需重启）**

```bash
sudo nano /etc/environment
# 文件内添加配置（无需写export，每行格式 key="value"）
MY_GLOBAL_VAR="从environment加载"
# 保存退出后需重启系统生效
```

**方法二：/etc/profile（所有用户登录时执行）**

```bash
sudo nano /etc/profile
# 在文件末尾添加
export JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64
# 执行命令立即生效（当前终端）
source /etc/profile
```

**方法三（推荐）：/etc/profile.d/ 目录新增 .sh 脚本**

```bash
# 创建自定义环境变量脚本
sudo nano /etc/profile.d/myenv.sh
# 脚本内添加配置
export MY_GLOBAL_VAR="global var"
export ADMIN_EMAIL="admin@example.com"
# 保存退出

# 执行命令立即生效（当前终端）
source /etc/profile.d/myenv.sh

# 切换到普通用户验证全局变量是否生效
su - testuser
echo $MY_GLOBAL_VAR   # 输出：global var（验证成功）
```

## 5. 知识总结

### 5.1 核心配置文件汇总

| 配置文件 | 作用 | 作用范围 | 加载时机 |
|----------|------|----------|----------|
| `/etc/passwd` | 存储用户账户基础信息（用户名、UID、GID、主目录、Shell） | 系统全局 | 用户登录、权限校验时读取 |
| `/etc/shadow` | 存储用户加密密码信息（仅root可读） | 系统全局 | 用户身份认证时读取 |
| `/etc/group` | 存储系统用户组信息 | 系统全局 | 组权限校验时读取 |
| `/etc/gshadow` | 存储用户组加密密码信息 | 系统全局 | 组身份认证时读取 |
| `~/.bashrc` | 用户Shell终端个性化配置（别名、函数、变量） | 当前用户 | 每次打开新非登录终端时执行 |
| `~/.profile` | 用户登录环境初始化配置 | 当前用户 | 用户登录系统时执行一次 |
| `~/.bash_logout` | 用户退出Shell时执行的清理脚本 | 当前用户 | 退出登录Shell时执行 |
| `/etc/profile` | 系统全局登录环境配置 | 所有用户 | 任意用户登录时执行 |
| `/etc/profile.d/*.sh` | 系统模块化环境配置脚本 | 所有用户 | 用户登录时按文件名顺序执行 |
| `/etc/environment` | 系统全局环境变量（纯键值对，不支持脚本语法） | 所有用户 | 系统启动时加载，需重启生效 |

### 5.2 环境变量加载顺序

登录Shell的环境变量加载顺序（Ubuntu Server默认）：

```
系统启动 → /etc/environment → /etc/profile → /etc/profile.d/*.sh → ~/.profile → ~/.bashrc
```

**优先级规则**：后加载的配置会覆盖先加载的同名变量，其中 `~/.bashrc` 具有最高优先级。

### 5.3 文件权限核心规则

- Linux将文件权限分为**所有者(u)**、**所属组(g)**、**其他用户(o)** 三类身份，每类身份独立拥有**读(r)**、**写(w)**、**执行(x)** 三种权限。
- **数字权限**：三位八进制数，分别对应三类身份的权限值之和（r=4，w=2，x=1），例如 `755` 代表所有者拥有读写执行权限（7=4+2+1），所属组与其他用户拥有读+执行权限（5=4+1）。
- **常用权限组合**：

| 权限数字 | 含义 | 适用场景 |
|----------|------|----------|
| 644 | 所有者读写，组和其他只读 | 普通文本文件 |
| 755 | 所有者读写执行，组和其他读执行 | 可执行程序、目录 |
| 600 | 仅所有者读写 | 私密配置文件（如SSH私钥） |
| 700 | 仅所有者读写执行 | 私有目录 |

### 5.4 常用Linux命令速查表

| 分类 | 命令示例 | 说明 |
|------|----------|------|
| 文件操作 | `ls, cd, cp, mv, rm, mkdir, touch` | 浏览、复制、移动、删除、创建 |
| 文本查看 | `cat, less, head, tail, grep` | 查看和搜索文本内容 |
| 权限管理 | `chmod, chown, chgrp, umask` | 修改权限和所有者 |

| 进程 | `ps, top, kill, pkill` | 查看和控制进程 |
| 软件包 | `apt update, apt install, apt remove` | 安装和卸载软件 |
| 系统 | `systemctl, reboot, shutdown, free, df` | 控制服务和查看资源 |


## 6. 出现问题及解决方案

| 问题现象 | 可能原因 | 解决方法 | 是否解决 |
|----------|----------|----------|----------|
| 执行 `sudo` 命令提示 `sudo: command not found` | 当前用户未加入 sudo 用户组 | 切换到 root 用户（`su -`），执行 `usermod -aG sudo puretaoist`，重新登录后生效 | ✅ |
| 修改文件或执行命令提示 `Permission denied` | 当前用户对目标文件/目录无对应操作权限 | 使用 `sudo` 提权执行命令，或通过 `chmod` 修改目标文件的访问权限 | ✅ |
| 修改 `~/.bashrc` 后配置的环境变量不生效 | 未手动重载配置文件，当前终端未加载新配置 | 执行 `source ~/.bashrc` 手动重载，或重新打开终端 | ✅ |
| 修改 `/etc/environment` 后环境变量不生效 | 该配置文件仅在系统启动时加载，修改后未重启 | 重启 Ubuntu 系统（`sudo reboot`），配置将在启动时全局生效 | ✅ |
| 执行 `ls -l` 显示文件权限为 `-rw-r--r--` 但无法修改 | 文件所在目录无写权限（父目录权限限制） | 检查上级目录写权限：`chmod u+w /path/to/dir` | ✅ |
| `adduser` 与 `useradd` 命令混淆 | 两个命令都可用于创建用户，但行为不同 | `adduser`（交互式，推荐），`useradd`（低层命令，需手动指定参数） | ✅ |


## 7. 心得体会

### 技术层面

通过本次实验，系统掌握了Linux命令行的基础操作逻辑，理解了Linux **“一切皆文件”** 的设计思想——无论是普通文本、目录、设备还是进程信息，均以文件形式暴露给用户。通过实际操作，我理清了用户权限分级管理的规则（所有者、所属组、其他用户三级隔离），理解了用户级（`~/.bashrc`、`~/.profile`）与系统级（`/etc/profile`、`/etc/environment`）环境变量的加载逻辑与配置差异。

实验中最印象深刻的是环境变量的 **加载顺序与优先级机制**——从系统启动到用户登录，多个配置文件依次加载，后加载的配置覆盖先加载的同名变量。这一设计既保证了系统级配置的统一性，又赋予用户级配置充分的灵活性，体现了Linux在多用户场景下的精巧设计。

此外，通过 `chmod` 数字法与符号法的对比练习，我深刻体会到Linux权限体系的严谨与高效——仅用三位八进制数即可精确控制九种权限组合，这种简洁的设计哲学贯穿整个操作系统。

### 思维层面

Linux作为开源操作系统的代表，充分体现了开源社区的共享协作精神。全球数以万计的开发者共同维护内核与软件包，使得系统具备极高的灵活性与安全性。Debian系发行版通过 `apt` 包管理器可以随时获取来自全世界的开源软件，这种高效的协作模式让我对开源生态有了更直观的认知。

对比国内外开源生态，国内Linux发行版（如统信UOS、麒麟OS）正快速发展，但在生态完善度与社区活跃度上仍有提升空间。我们应当积极参与开源社区建设，夯实基础技术能力，从使用开源到回馈开源。同时，命令行操作需要高度的精确性——一个空格、一个斜杠的错误都可能导致完全不同的结果，这对开发者有严格的 **严谨的工程态度** 的要求。
