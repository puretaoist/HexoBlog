## 一、实验目的

本次实验旨在安装 Visual Studio Code 代码编辑器，安装和配置 Markdown 相关插件，掌握 VS Code 的基本界面布局和插件扩展机制，并使用 Markdown 语法完成文档的编辑和实时预览，为后续软件开发工作奠定高效的编辑基础。

## 二、实验环境

- 操作系统：Windows 11 专业版 23H2
- Visual Studio Code：1.85.0
- VS Code 插件：
  - Markdown All in One（v3.6.2）
  - Markdown Preview Enhanced（v0.8.14）
  - Paste Image（v1.0.4）
- 网络：需互联网连接以下载插件

## 三、实验过程

### 步骤一：安装 Visual Studio Code

访问 VS Code 官网，双击安装包启动安装向导，同意许可协议后进入安装路径选择，在"选择附加任务"页面，勾选以下选项：

1. 创建桌面快捷方式
2. 将"通过 Code 打开"操作添加到 Windows 资源管理器文件上下文菜单
3. 将"通过 Code 打开"操作添加到 Windows 资源管理器目录上下文菜单
4. 将 Code 注册为受支持的文件类型的编辑器
5. 添加到 PATH

### 步骤三：安装 Markdown 相关插件

打开扩展面板：点击左侧活动栏中的扩展图标（或使用快捷键 Ctrl + Shift + X）

在扩展面板搜索框中依次搜索以下插件并点击"安装"：

| 插件名称 | 功能 |
| --- | --- |
| Markdown All in One | 目录生成、快捷键支持、列表自动延续 |
| Markdown Preview Enhanced | 数学公式（LaTeX）、流程图（Mermaid）、导出为 PDF/HTML |
| Paste Image | 直接粘贴剪贴板中的图片到 Markdown 文件 |
| markdownlint | 检查 Markdown 语法错误和格式规范 |

### 步骤四：配置编辑器优化写作体验

按 Ctrl + , 打开设置，进行以下配置：

| 配置项 | 搜索关键词 | 设置值 |
| --- | --- | --- |
| 自动保存 | files.autosave | afterDelay |
| 单词折行 | editor.wordwrap | on |
| 图片粘贴路径 | Paste Image: Path Type | relative |

## 五、知识总结

### 1、代码编辑器 vs 集成开发环境（IDE）

| 类型 | 特点 | 典型代表 |
| --- | --- | --- |
| 代码编辑器 | 轻量级、启动快、以编辑为核心、通过插件扩展能力 | VS Code、Sublime Text |
| 集成开发环境 | 功能全面、集成了编译/调试/构建/版本控制等全套工具链 | IntelliJ IDEA、PyCharm |

### 2、VS Code 插件扩展机制

插件是 VS Code 最核心的扩展机制。通过安装不同插件，可以为编辑器增加语法高亮、代码补全、格式化、预览、调试等新功能。

### 3、Markdown 核心语法

Markdown 是一种轻量级标记语言，通过简单的纯文本语法实现结构化文档编写，并支持转换为 HTML、PDF 等格式。

**核心特点：**

- **简洁性**：仅需少量符号（如 `#`、`*`、`-`）即可完成排版
- **可读性**：未渲染的纯文本仍然清晰易读
- **兼容性**：支持 GitHub、各大博客平台、笔记工具等场景

## 六、出现问题

| 问题现象 | 可能原因 | 解决方法 |
| --- | --- | --- |
| 安装后无法在命令行使用 `code` 命令 | 安装时未勾选"添加到 PATH" | 重新运行安装程序，勾选"添加到 PATH"；或手动将 VS Code 安装目录添加到系统环境变量 |
| 扩展面板搜索不到插件 | 网络问题或搜索关键词不准确 | 检查网络连接；尝试使用准确的插件全名搜索 |
| 预览窗口不显示或显示异常 | 插件未正确安装或未启用 | 检查扩展面板中插件是否已启用；重启 VS Code |
