## 1. 实验目的

本次实验旨在掌握虚拟机技术的核心概念与基本操作，使用 Oracle VirtualBox 在 Windows 宿主机上安装 Ubuntu Server LTS 操作系统，并配置双网卡（NAT + 仅主机）网络环境。通过模板机克隆快速构建多节点，理解虚拟化环境下的网络隔离与互联机制，为后续搭建 Linux 开发环境奠定基础。

## 2. 实验环境

- **宿主机操作系统**：Windows 11 专业工作站版 23H2  
- **虚拟化软件**：Oracle VirtualBox 7.0.12（需 CPU 支持 VT-x/AMD-V）  
- **客户机操作系统**：Ubuntu Server LTS（ISO 镜像）  
- **硬件配置**：Intel u7 255hx / 16GB RAM / 1TB SSD（分配 2GB 内存 / 20GB 磁盘每台虚拟机）  
- **网络拓扑**：  
  - 适配器1：NAT（用于访问外网）  
  - 适配器2：Host-Only（虚拟网段 192.168.56.0/24，宿主机 IP 为 192.168.56.1）  
- **虚拟机规划**：模板机（template）及克隆机 node1、node2、node3，IP 分别为 192.168.56.101~104。截图为个人实验，可能会与教程有命名出入

### 步骤4：安装 Ubuntu Server

启动虚拟机，进入安装界面：
1. 选择语言 **English**，键盘布局 **English (US)**。
2. 网络配置阶段，系统自动识别 NAT 网卡（enp0s3）并获取 DHCP 地址，直接选择 “Done”。
3. 代理配置留空，镜像地址保持默认。
4. 存储配置：选择 **Use an entire disk**（使用整块磁盘），确认分区方案后选择 “Done” → “Continue”。
5. 设置用户信息：
   - 用户名：`student`(例子)
   - 主机名：`ubuntu`（可暂留，后续修改）
   - 密码：设置强密码（如 `P@ssw0rd`）
6. **关键步骤**：在 “SSH server” 页面，按空格键选中 **Install OpenSSH server**，以便后续远程管理。
7. 等待安装完成（约 3~5 分钟），重启虚拟机。

### 步骤5：登录并配置静态 IP（双网卡）

重启后输入用户名 `student` 和密码登录。查看网卡名称：
```bash
ip a
```
通常 NAT 网卡为 `enp0s3`，仅主机网卡为 `enp0s8`。编辑 Netplan 配置文件：
```bash
sudo nano /etc/netplan/50-cloud-init.yaml
```
修改为以下内容（注意缩进）：
```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: yes
    enp0s8:
      dhcp4: no
      addresses:
        - 192.168.56.101/24
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114]
```
保存（Ctrl+O，回车，Ctrl+X），应用配置：
```bash
sudo netplan apply
```
验证：`ping -c 3 8.8.8.8`（外网）和 `ping -c 3 192.168.56.1`（宿主机）。

### 步骤6：修改主机名与 hosts 解析

将主机名改为 `template`：
```bash
sudo hostnamectl set-hostname template
```
编辑 hosts 文件，加入所有节点解析：
```bash
sudo nano /etc/hosts
```
添加：
```
127.0.0.1 localhost
192.168.56.101 template
192.168.56.102 node1
192.168.56.103 node2
192.168.56.104 node3
```
保存后，退出重新登录即可看到主机名变化。

### 步骤7：克隆模板机

关闭模板机：`sudo shutdown now`。在 VirtualBox 管理器中右键点击 `ubuntu-template`，选择 “克隆”(即复制)：
- **克隆类型**：选择 **Full clone**（完整克隆）或 **Linked clone**（链接克隆，本次选完整克隆以保证独立性）
- **名称**：填写 `node1`
- **MAC 地址策略**：选择 **为所有网卡重新生成 MAC 地址**（避免冲突）
- 点击完成，等待克隆结束。重复此操作分别克隆出 `node2`、`node3`。

### 步骤8：修改克隆机的主机名和 IP

分别启动 node1、node2、node3，登录后：
- 修改主机名（以 node1 为例）：
  ```bash
  sudo hostnamectl set-hostname node1
  ```
- 修改 Netplan 文件，将 `192.168.56.101` 改为对应 IP（node1→102，node2→103，node3→104）：
  ```bash

  # 修改 addresses 下的 IP 地址
  sudo netplan apply
  ```
- 更新 hosts 文件（可选，若克隆时已包含则无需再改）。

## 5. 知识总结

### 5.1 虚拟机网络模式对比

| 模式 | 虚拟机能否访问外网 | 外部能否访问虚拟机 | 宿主机能否访问虚拟机 | 典型应用场景 |
|------|-------------------|-------------------|----------------------|--------------|
| **NAT** | ✅ 是（通过宿主机转发） | ❌ 否（除非端口转发） | ✅ 是 | 安装软件、更新系统 |
| **桥接（Bridge）** | ✅ 是（独立 IP） | ✅ 是（同网段可直接访问） | ✅ 是 | 提供对外服务（如 Web 服务器） |
| **仅主机（Host-Only）** | ❌ 否 | ❌ 否 | ✅ 是 | 搭建内部实验网络，多机互联 |

本次采用 **NAT + 仅主机** 双网卡，既满足了联网需求，又实现了宿主机与虚拟机、虚拟机之间的稳定内网通信。

### 5.2 Netplan 网络配置要点

- Netplan 是 Ubuntu 18.04+ 的默认网络配置工具，YAML 格式。
- 关键字段：
  - `dhcp4: yes/no` —— 是否启用 DHCP
  - `addresses` —— 静态 IP 和掩码（CIDR 格式）
  - `nameservers` —— DNS 服务器列表
- 应用配置：`sudo netplan apply`
- 调试：`ip a`、`ping`、`route -n` 查看路由

### 5.3 克隆虚拟机注意事项

- **MAC 地址**：必须重新生成，否则网络冲突。
- **主机名**：克隆后需手动修改，否则无法区分节点。
- **IP 地址**：需修改静态 IP，避免重复。
- **SSH Host Key**：若启用 SSH，可能会因重复主机密钥导致安全警告，建议删除 `/etc/ssh/ssh_host_*` 并重新生成（本次实验未出现异常）。

### 5.4 常用 Linux 管理命令

| 命令 | 用途 |
|------|------|
| `sudo hostnamectl set-hostname <新名>` | 修改主机名 |
| `sudo netplan apply` | 应用网络配置 |
| `ping -c <次数> <目标>` | 测试连通性 |
| `ssh student@<IP>` | 远程登录 |
| `sudo shutdown now` | 立即关机 |
| `ip a` | 查看所有网络接口地址 |

## 6. 出现问题及解决方案

| 问题现象 | 可能原因 | 解决方法 | 是否解决 |
|----------|----------|----------|----------|
| 新建虚拟机时提示“VT-x/AMD-V 未启用” | BIOS 中虚拟化功能未开启 | 重启电脑进入 BIOS，开启 Intel VT-x 或 AMD SVM | ✅ |
| 安装 Ubuntu Server 时卡在“网络配置” | DHCP 未获取到地址（NAT 网卡未启动） | 检查 VirtualBox 网络设置中 NAT 适配器是否启用；尝试重启虚拟机 | ✅ |
| 配置完静态 IP 后 `ping 8.8.8.8` 不通 | DNS 未配置或网关缺失 | 在 netplan 中正确添加 `nameservers`，并检查 NAT 网卡 `dhcp4: yes` 自动获取网关 | ✅ |
| 宿主机无法 ping 通虚拟机（192.168.56.101） | 宿主机 Host-Only 网卡未启用或 IP 不在同一网段 | 检查 Windows 网络适配器中的 VirtualBox Host-Only Network，确保 IP 为 192.168.56.1，掩码 255.255.255.0，如果不是可在virtualbox网络一栏修改 | ✅ |
| 克隆后 node1 启动，网络无法连接（`Failed to start Network`） | 克隆时未重新生成 MAC 地址，导致网卡冲突 | 在 VirtualBox 虚拟机设置 → 网络 → 高级 → 刷新 MAC 地址；启动后重新配置 netplan | ✅ |
| 克隆机之间 ssh 登录提示“Host key verification failed” | 克隆机与模板机共享相同的 SSH 主机密钥 | 登录克隆机，执行 `sudo rm /etc/ssh/ssh_host_*`，然后 `sudo dpkg-reconfigure openssh-server` 重新生成密钥 | ✅ |
| 测试ping显示100%loss | 克隆的虚拟机网络中网卡没有设置NAT和仅主机俩个网卡 | 在虚拟机网络中启动网卡 | ✅ |

## 7. 心得体会

**技术层面**：  
通过本次实验，深入了解了虚拟机技术的实现原理，掌握了 VirtualBox 的基本操作——从创建虚拟机、安装操作系统、配置双网卡，到克隆和批量配置多节点环境。Netplan 的配置展示了Linux 网络管理的部分内容，尤其是静态 IP 与 DHCP 的混合使用场景。克隆功能的便捷性使得搭建多节点集群变得高效。同时认识到无gui面向服务端的系统及其命令行操作，初步了解Linux服务器面貌。

**思维层面**：  
虚拟化技术极大降低了硬件成本和实验门槛，能够在一台笔记本上模拟多台服务器，这体现了“软件定义基础设施”的先进理念。同时，配置网络时的严谨性（IP 规划、子网掩码、MAC 地址）意味着，任何一个细微的错误都可能导致整个集群通信失败，因此规范操作和文档记录至关重要。

**思政融合**：  
Ubuntu Server 作为开源操作系统，免费且功能强大，VirtualBox 同样开源，二者的结合使得全球无数开发者能够低成本地学习和实验。这属于开源社区对技术普及和教育公平的巨大贡献。此外，在配置 SSH 和网络安全方面，设置强密码、管理密钥十分重要——网络技术的每一步都要兼顾便捷与安全，这是信息技术工作者的基本职业素养。
